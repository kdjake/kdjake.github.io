SurveyMonkey -Data Transformation

Project Overview

This project involved cleaning and analyzing a survey dataset using Python's Pandas library. The skills demonstrated include importing data from Excel into Pandas, renaming columns, unpivoting columns (also known as melting), joining columns, aggregating columns, and dropping columns. The dataset used was SurveyMonkey, consisting of over 30 questions with varying numbers of answer choices for each question. The dataset was in a wide format in an Excel spreadsheet, which made it difficult to analyze the data effectively in Tableau. Thus, the dataset was transformed into a long format.

Prior to the data transformation process, the dataset was cleaned by removing any irrelevant or duplicate information, such as rows with missing values or duplicated responses. Additionally, inconsistencies or errors in the data were checked and corrected. Finally, the dataset was analyzed to answer specific research questions, such as "How many people answered a question per given question" and "How many people answered the same answer per question."                     

Process

1: Initial Excel 


In order to analyze the survey data, I performed several data preparation steps. First, I merged the first two rows to create a single header row that contained the responses for each individual in separate columns. To ensure that the dataset was complete, I also used IF statements, transpose and concatenate functions to fill in any blank cells.

As a result of this data transformation process, I was able to create a new "Question + Sub-question" column that contained the complete question text for each survey response. This column served as the single header for the dataset, providing a clear and concise overview of the survey responses for each individual.

After Excel cleaning

Next, I proceeded to perform further data cleaning and analysis using Python. After importing the necessary libraries, I utilized the pwd (present working directory) function to facilitate the file-moving process and streamline the data cleaning process.
By using pwd, I was able to easily navigate through my file system and locate the files I needed for my analysis. This not only saved time, but also ensured that the correct files were being used for analysis.



To ensure that I could easily correct any mistakes, I made a copy of the dataset using the code "dataset_modified = dataset.copy()". This approach allows me to revert back to the original dataset if I make a mistake during the cleaning or analysis process.
This is particularly useful when working with large datasets, as importing the data repeatedly can be a time-consuming process. By creating a copy of the original dataset, I am able to make changes and perform analysis without worrying about accidentally overwriting the original dataset. Next, I proceeded to delete any empty columns in the dataset.



The next step in preparing the dataset for analysis was to unpivot the data using the melt method. This approach involved converting the wide format of the dataset into a long format, where each row represents a unique observation and each column represents a variable.

I then imported the "questions" sheet and dropped any columns that were not going to be used in the analysis. Since this sheet was only used as a reference to create the header row for the dataset, there were some columns that were not necessary for the analysis. To remove these unnecessary columns, I used the drop method and specified the columns to be removed. Additionally, I used the ‘inplace=True’ parameter to ensure that the changes were made to the dataset.


At this stage, I had two datasets - the questions dataset and the melted survey dataset. To correlate the question and response data based on the question, I performed a join to merge these two datasets together. By using the question column as a common key, I was able to match the survey responses to their corresponding questions, as one question could have multiple responses. This enabled me to link the responses to their specific questions. 



Next, I performed aggregation to answer the question "How many people answered each question," as not every individual responded to every question. We used the ‘groupby’ function and as we can see from the result, a count of 198 means that everyone answered that particular question. 


Next, I performed the second merge by joining the two datasets "dataset_merged" and "respondents" on the question column. This allowed me to visualize the number of respondents for each question.



The next question I needed to answer was "How many people provided the same answer for each question?" To do this, I used the ‘groupby’ function again and applied the count method to the response column. The results indicate that there were 688 distinct answers that could have been given. After obtaining the dataset of the number of people who gave the same answer per question, I proceeded to merge it with the 'dataset_merged_two' using the join function in Python.

After finishing the data cleaning and analysis, the final step was to assign more readable and understandable names to the column headers. Once the renaming was completed, I downloaded the final edited Excel file to be used for further analysis and visualization in Tableau.


Final Edited Excel 



The final edited excel file was downloaded, which can now be easily imported into Tableau for further visualization and analysis. By transforming the dataset into a long-format, it is now easier to understand the survey results and draw meaningful insights from the data. Overall, this project demonstrated the importance of proper data preparation and manipulation in order to effectively analyze and visualize data.
